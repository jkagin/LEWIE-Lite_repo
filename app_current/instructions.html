WELCOME to the LEWIE-Lite DASHBOARD

On this dashboard, you can run simple simulations to calculate multipliers of a tourism project using a LEWIE-Lite model.

A LEWIE-Lite model is a multiplier model based on a Social Accounting Matrix (SAM) of the target area.

+ What does the Dashboard display?

The dashboard is divided into two main panels:

The HOME tab

It shows basic information about the park, along with tourism multipliers.

The top part presents some of the park information gathered in the process of constructing the LEWIE-lite model.

It also allows you to run quick calcualtions of park revenues if tourism were to increase or decrease.

The bottom part shows multipliers of tourism activities in the project area. These are the key outputs of the LEWIE-lite model. They represent dollar increases per dollar of tourist spending.

The Total production multiplier is the overall increase value generated in the economy (any goods or services) for each dollar spent by a tourist.

The Total income multiplier is the dollar increase in household incomes for each dollar of tourist spending. We also break down this income multiplier in two ways:

  • The total income multiplier can be broken down into:
    • Accruing to Poor households: Dollar increase in Poor household incomes for each dollar of tourist spending.
    • Accruing to NonPoor households: Dollar increase in NonPoor household incomes for each dollar of tourist spending.
  • The same total income multiplier can also be broken down into:
    • Accruing to Labor: Dollar increase in household incomes from use of their Labor (for each dollar of tourist spending)
    • Accruing to Capital: Dollar increase in household incomes from as a return to Capital (for each dollar of tourist spending)

The SIMULATIONS Tab

It containc a menu list at the top, followed by a series of panels showing plots that can be used in simulations. There are 9 panels, each showing what happens if a given monetary value ($) is injected into the economy. The menu at the top of each section enable quick navigation between the simulation panels. These are based on the multipliers, but can be scaled by changing tourist spending.

Taking the first panel as an example (Tourist Spending), the simulation panels are composed of:

  • An input box. E.g., "How much tourist spending ($) do want to simulate?". This is set to $100 by default. It can be changed to any positive value, and the expected earnings will be shown in the plots. (Note that this does not change the multipliers, which are all per-dollar values. Rather, it is based on the multipliers).
  • Several plots. The three plots show the returns on Production (split by activity), Household Incomes (split by household type) and Labor Income (split by gender and skill), which would be generated by an injection of the given the monetary value in the input box. The Tourism Spending panel has a fourth panel showing how tourism affects park revenue (not relevant for simulations other than tourist spending)

All panels have similar elements. All plots are based on the same multipliers as those displayed in the headboard.

Simulations are performed by changing the value in the input box. These are separate for each simulation panel.

+ Can I download these results?

There is a "Generate Report" button at the bottom of the Home page.

This process is slow. Please allow the server some time to generate the report.

+ What is a multiplier?

A multiplier captures the fact that money circulates through the economy. The production multiplier represents the total value of goods and services generated in the economy for every dollar spent by a tourist, including all higher-order impacts, or ripple effects (e.g.: Tourist spends money at a restaurant, the cook spends some of those wages at a local shop, the shopkeeper spends some of those earnings buying food from a farmer, etc.)

The income multiplier is the same concept, but only counts household incomes. It is always smaller than the production multiplier, because only a fraction of the production value turns into income (i.e. you have to net out the costs)

+ How to use the Dashboard?

The first way to use the dashboard is simply to read and report the multipliers in the header (top panel).

The second way to use it is to input any value you want under "How much [...] ($) do want to simulate?" to change the values displayed in the plots. These values correspond to the multipliers for each category. See below for more information.

+ What do the simulations mean?

The simulations show the value generated in different sectors of the economy by an increase in spending equal to the value in the input box.

Technically speaking, each simulation is based on a column from the multiplier matrix. The plots are generated by multiplying the value in the box by the relevant entries in the multiplier matrix.

Example: Tourism spending

  • By default, it show the impact of 100$ of additional tourism spending. This can be changed to any value $V
  • The first plot shows how much $V of tourism spending generates in various activities, related to tourism or not. The value accounts for the higher-order impacts of tourism spending. For instance, although tourists may not purchase vegetables directly from local farmers, the hotel staff might. Such secondary impacts are accounted for by the multiplier model.
  • The second plot shows the amounts generated as incomes of local households as a result of spending $V. Again, this accounts for direct incomes (e.g. the hotel owner's income) and indirect incomes (e.g. the local farmer who benefits indirectly.)
  • The third plot shows the amounts generated specifically as labor income (not capital income). As many tourism-related activities are capital-intensive (e.g. hotels) these amounts will generally be smaller than those in the second plot.
  • The fourth plot shows park and community revenues. These values depend heavily on local arrangements and rules about how park fees get spent.

+ Can I see the SAM?

All multipliers are based on the SAM displayed in the SAMs panel. The stages of constructing that SAM are also shown: "SAM_preRAS" refers to the the closest we could get to a balanced matrix using the raw data inputs. "SAM_RASed" refers to the the output of a "RAS" balancing procedure (bi-proportional balancing) that was performed with the pre-RAS matrix as the input. "Shares" refers to the column shares of the RASed matrix.

The DATA tab shows all the data that went into constructing the SAM (see below)

+ What is the DATA Tab?

The Data tab shows all the inputs and outputs of the model.

The data input panel shows all the values used to generate the model.

The bottom panel has three tabs:

  • A SAM_RASed tab. This shows the SAM used to generate the multipliers. "RAS" refers to a SAM-balancing procedure.
  • A SAM_preRAS tab. This shows an unbalanced version of the SAM and is primarily there for debugging purposes.
  • A Multipliers tab. This shows the multiplier matrix generated from the SAM_RASed tab. All the results presented in the dashboard can be traced back to this multiplier matrix.
